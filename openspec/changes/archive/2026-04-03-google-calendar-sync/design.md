# Design: Google Calendar Integration

## Architecture

The integration follows a **Service + Observer** pattern to maintain a decoupled architecture.

### Schema Transformation
- **`CompanyProfile`**: Stores `google_calendar_credentials` as an encrypted `TextField`. This avoids exposing private JSON keys on the database level.
- **`Booking`**: Stores `google_event_id` (managed internally). This id is generated by the Google API on the first successful sync.

### Service Layer (`GoogleCalendarService`)
This layer encapsulates the `google-api-python-client`.
- **Authentication**: Uses `google.oauth2.service_account` to read the credentials and sign requests.
- **Sync Logic**:
  - `SYNC (Booking)`:
    - If `google_event_id` is null → `service.events().insert(...)`
    - Else → `service.events().patch(...)`
- **Error Handling**: Gracefully handle missing calendars or invalid credentials without interrupting the application flow.

### Automation via Signals
Instead of overriding the `save()` method on `Booking` (which would make testing and batch operations harder), we will use Django's `post_save` signals.
- **Trigger**: When a `Booking` is created or updated.
- **Condition**: Only sync if the `CompanyProfile` has both `google_calendar_id` and `google_calendar_credentials` configured.

### Production Readiness
- **Performance**: While synchronously calling the API in a signal is acceptable for MVP, the design should facilitate an easy move to **Celery** background tasks.
- **User Permissions**: The design assumes **Service Account** integration. Tenants must grant the service account's email "Make changes to events" access in their Google Calendar.

## Trade-offs
- **Service Account vs User OAuth**: Service accounts are chosen as they are easier to implement of a server-to-server context and provide stability without refresh-token management overhead for the tenant once configured.
- **Synchronous API calls**: Chosen for MVP to avoid adding Celery complexity immediately, but with clear abstraction to allow future async execution.
